# China Policy Risk Analysis Skill

一个面向普通居民视角的中国政策与宏观金融风险分析 Skill。它用于阅读公开政策文本、统计数据和配套规则时，帮助识别政策目标、传导链条、执行约束、数据一致性、潜在风险承担方，以及普通居民需要关注的风险控制点。

本 Skill 的定位是审慎分析工具，不是立场表达工具。它强调从公开信息出发，把政策影响拆解为可观察、可验证、可复盘的路径。

## 适用场景

- 分析财政、货币、金融、房地产、资本市场、养老、社保、产业、地方债、平台经济等政策。
- 判断政策可能影响哪些机构、行业或群体，以及成本可能通过哪些渠道分摊。
- 评估政策落地难度、执行偏差、配套政策缺口和后续可能补充的工具。
- 分析官方统计数据时，拆分细项、识别口径限制、估算可能偏差，并进行交叉印证。
- 从普通居民视角评估就业、储蓄、负债、房产、养老金、基金、保险和其他金融产品风险。
- 分析宏观金融压力在人民币汇率、房地产、地方债、银行、就业收入、资本流动和资本市场之间的传导。

## 分析原则

1. 不把政策文本等同于政策结果。先看政策说什么，再看资金从哪里来、风险往哪里去、谁有退出权。
2. 不只看受益方，也看成本承担方和最终风险承接方。
3. 不只看主政策，也看配套政策、实施细则、资金来源、监管规则和地方执行能力。
4. 不只看官方数据大类，也要看细分项、统计口径、时间滞后、区域差异和交叉验证结果。
5. 不把“政策支持”理解为个人收益保证。系统性稳定工具通常优先保护系统流动性，不等于保证个人资产不亏损。
6. 普通居民建议默认采取审慎立场：先控制不可逆损失，再讨论可能的政策红利。

## 标准分析框架

使用本 Skill 时，建议按以下结构输出：

1. **核心判断**：给出一句话结论，并标注政策落地风险。
2. **政策表层与现实约束**：区分官方表述、现实压力和可能的制度约束。
3. **官方数据校验**：拆分关键细项，评估数据可靠性、可能偏差和交叉印证结果。
4. **传导链条**：说明政策如何影响金融机构、企业、地方政府和居民部门。
5. **受益方、受损方、风险承接方**：避免只写利好，不写成本。
6. **配套政策与缺口**：列出已有配套；若缺失，分析未来可能补哪些工具及副作用。
7. **短期影响**：关注流动性、资产价格、就业预期、融资成本和市场情绪。
8. **长期后果**：关注债务、财政、产业竞争、居民财富结构和制度约束。
9. **普通居民行动框架**：先写风险规避，再写可能机会。

## 官方数据校验

涉及官方统计、监管披露或地方数据时，默认加入以下检查：

- 不只看总量，也看分项、区域、所有制、价格和数量拆分。
- 对可能存在的统计偏差或“水分”给出范围，而不是给出单点结论。
- 用财政收入、用电、货运、房屋成交、企业财报、市场价格、贸易伙伴数据、居民行为等进行交叉印证。
- 当数据互相矛盾时，解释可能原因：口径变化、时间滞后、结构分化、价格数量错配、样本偏差或政策平滑。

## 普通居民风险提示

本 Skill 对普通居民的建议默认采取保守立场：

- 保留足够现金流，不为政策主题加杠杆。
- 不把理财、基金、保险、养老金产品等同于存款。
- 警惕“稳健”“养老”“政策支持”“高股息”“固收+”等营销标签。
- 长债基金、信用债基金、红利资产、主题基金都可能出现本金回撤。
- 个人养老金账户需要关注锁定期、税收适配度、产品费用和未来规则变化。
- 房产需要区分自住和投资，政策托底不等于价格上涨或流动性恢复。
- 对复杂、封闭、难退出、收益承诺模糊的产品保持克制。

## 使用方式

在 Codex 中可以这样调用：

```text
Use $china-policy-risk-analysis to analyze this Chinese policy's implementation risk, official-data consistency, transmission chain, and ordinary resident risk controls.
```

也可以直接提出中文请求，例如：

```text
使用 china-policy-risk-analysis 分析这项政策对普通居民、银行、企业和地方政府的影响，并评估官方数据是否存在需要交叉验证的矛盾点。
```

## 审慎边界

本 Skill 仅基于公开资料和一般政策分析方法提供结构化研判，不提供个性化投资建议、法律意见或规避监管建议。对于金融产品、房产、养老金账户等事项，使用者仍需结合自身现金流、负债、家庭责任、风险承受能力和当地规则独立判断。
